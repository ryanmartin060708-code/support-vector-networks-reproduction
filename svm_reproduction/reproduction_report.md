# Reproduction Report: Support-Vector Networks

## 1. Summary of Paper

Cortes and Vapnik introduce support-vector networks as maximum-margin classifiers
that operate in high-dimensional feature spaces through kernels. The paper
extends optimal hyperplanes from separable data to non-separable data using soft
margins and demonstrates that polynomial kernels up to degree 7 can generalize
well on handwritten digit recognition.

The central empirical claim is that high-dimensional polynomial decision
surfaces do not necessarily overfit when the solution is controlled by the
margin and a relatively small support-vector set.

## 2. Mathematical Derivation

The hard-margin SVM solves

```text
minimize    1/2 ||w||^2
subject to  y_i (w^T x_i + b) >= 1.
```

The Lagrangian stationarity conditions imply

```text
w = sum_i alpha_i y_i x_i
sum_i alpha_i y_i = 0.
```

The dual is

```text
maximize    sum_i alpha_i
            - 1/2 sum_i sum_j alpha_i alpha_j y_i y_j K(x_i, x_j)
subject to  alpha_i >= 0
            sum_i alpha_i y_i = 0.
```

For the requested L1 soft-margin objective,

```text
minimize    1/2 ||w||^2 + C sum_i xi_i
subject to  y_i (w^T x_i + b) >= 1 - xi_i
            xi_i >= 0,
```

the dual adds the box constraint

```text
0 <= alpha_i <= C.
```

KKT complementarity identifies support vectors: `alpha_i > 0` only when the
constraint is active. In the soft-margin case, `alpha_i = C` indicates a point
inside the margin or a misclassified point.

The kernelized prediction rule is

```text
f(x) = sum_{i in SV} alpha_i y_i K(x_i, x) + b.
```

The paper's main kernel is

```text
K(u, v) = (u^T v + 1)^d.
```

## 3. Implementation Details

The project implements the SVM pipeline from scratch:

- `src/kernels.py`: linear, polynomial, and Gaussian/RBF kernels.
- `src/optimizer.py`: dual objective, KKT diagnostics, SciPy QP wrapper,
  from-scratch SMO, and a paper-style chunking active-set wrapper.
- `src/svm.py`: binary hard-margin, binary soft-margin, and one-vs-rest
  multiclass estimators.
- `src/metrics.py`: accuracy, confusion matrix, precision, recall, and F1.
- `src/utils.py`: USPS/MNIST loading and OCR preprocessing.
- `src/visualization.py`: decision boundaries, support-vector plots, digit
  grids, and benchmark plots.

`sklearn.svm.SVC` is not used. SciPy is optional for exact small QP solves; SMO
is the main reusable implementation.

## 4. Experimental Setup

### Synthetic

The paper does not publish coordinates for Figure 5. The reproduction generates
analogous 2D datasets and trains degree-2 polynomial soft-margin SVMs. It marks
support vectors, margins, the decision boundary, and training errors.

### USPS

The USPS script downloads modern public USPS libsvm-format mirrors. It applies
centering, deslanting, and Gaussian smoothing with `sigma=0.75`, then trains ten
one-vs-rest polynomial classifiers for degrees 1 through 7.

The paper does not report the value of `C`; the script defaults to `C=10.0` and
exposes it as a command-line argument.

### NIST / MNIST

The original NIST split is not bundled. The reproduction uses MNIST as a
60,000/10,000 substitute, trains ten one-vs-rest degree-4 polynomial SVMs, and
reports accuracy, macro precision, macro recall, macro F1, and support-vector
counts. Because normalized MNIST pixels with the literal `(u^T v + 1)^4` kernel
produce very large Gram values, the MNIST script defaults to `gamma=0.01` for
the polynomial kernel. Passing `--gamma 1.0` recovers the literal paper kernel.

## 5. Results

Original reported results are transcribed in
`svm_reproduction/results/comparison.md`.

Reproduced result files generated by scripts:

- `results/synthetic_results.json`
- `results/usps_results.{json,csv,md}`
- `results/mnist_results.{json,csv,md}`

Full USPS/MNIST runs were left as executable scripts because they require
substantial compute. Smoke runs were executed on 20 examples per class to verify
the experiment code paths:

- USPS degree 1: 80.0% test accuracy, 23.2 mean support vectors.
- USPS degree 2: 81.5% test accuracy, 28.8 mean support vectors.
- MNIST degree 4: 82.0% test accuracy, 0.8195 macro F1, 685 total support
  vectors.

## 7. Failure Analysis

Expected sources of discrepancy:

- Exact synthetic coordinates are unavailable.
- The USPS mirrors are close to, but not exactly guaranteed to be, the paper's
  dataset split.
- The paper's `C` values are not reported.
- The paper's centering/deslanting algorithms are described semantically, not as
  executable pseudocode.
- MNIST is not the same dataset as the paper's NIST mixture.
- A pure Python SMO solver is much slower than the specialized solvers used in
  historical or modern production SVM systems.

## 8. Reproducibility Notes

All experiment scripts expose random seeds, solver tolerances, `C`, kernel
degree, and sample caps for smoke tests. Large runs should be executed in a fresh
environment after installing `requirements.txt`.
